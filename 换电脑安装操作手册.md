# 换电脑安装操作手册

## 1. 适用场景

这份手册适用于以下情况：

- 你换了一台新电脑，想把当前这套 Codex skills 恢复过去
- 你想在另一台机器上复刻同样的工作环境
- 你想给团队成员一份可执行的安装步骤

这份手册默认你已经：

- 安装好了 Codex
- 可以正常打开 Codex
- 当前机器可以联网访问 GitHub

## 2. 先决条件

建议先在新电脑上确认这些基础工具：

```bash
python3 --version
git --version
```

有些 skill 运行时还依赖 Node.js / npm，建议也一起确认：

```bash
node --version
npm --version
```

如果 `node` 或 `npm` 不存在，建议先安装 Node.js，因为下面这些 skill 会更依赖它：

- `playwright`
- `defuddle`
- `find-skills`

## 3. 获取这份仓库

先把本仓库拉到本地，方便随时对照：

```bash
git clone https://github.com/sun6762/installed_skills.git
cd installed_skills
```

也可以不克隆，直接在 GitHub 页面里对照文档操作。

## 4. 确认 Codex Skills 目录

Codex 的用户级 skills 通常安装在：

```bash
~/.codex/skills
```

你可以先确认目录存在：

```bash
mkdir -p ~/.codex/skills
ls ~/.codex/skills
```

## 5. 推荐安装顺序

建议按下面顺序安装：

1. `using-superpowers`
2. `playwright`
3. `markitdown`
4. `obsidian-markdown`
5. `obsidian-bases`
6. `obsidian-cli`
7. `json-canvas`
8. `defuddle`
9. `find-skills`
10. `security-threat-model`
11. `pdf`

这样做的好处是先把流程类和基础类 skill 装好，再装业务型 skill。

## 6. 安装方法

### 6.1 使用 Codex 自带的 skill-installer 脚本

如果你的 Codex 环境已经带有 `skill-installer`，优先用它安装。

脚本路径通常是：

```bash
/Users/你的用户名/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py
```

为了避免每次都写全路径，建议先设置一个变量：

```bash
export SKILL_INSTALLER="$HOME/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py"
```

然后就可以用下面的命令逐个安装。

---

### 6.2 安装 `using-superpowers`

```bash
python3 "$SKILL_INSTALLER" --url https://github.com/obra/superpowers/tree/main/skills/using-superpowers
```

---

### 6.3 安装 `playwright`

```bash
python3 "$SKILL_INSTALLER" --repo openai/skills --path skills/.curated/playwright
```

---

### 6.4 安装 `markitdown`

这个 skill 当前来自 K-Dense 仓库：

```bash
python3 "$SKILL_INSTALLER" --repo K-Dense-AI/scientific-agent-skills --path scientific-skills/markitdown
```

---

### 6.5 安装 Obsidian 系列 skills

这几个来自同一个仓库，可以一起装：

```bash
python3 "$SKILL_INSTALLER" \
  --repo kepano/obsidian-skills \
  --path skills/obsidian-markdown \
  skills/obsidian-bases \
  skills/json-canvas \
  skills/obsidian-cli \
  skills/defuddle
```

---

### 6.6 安装 `find-skills`

```bash
python3 "$SKILL_INSTALLER" --repo vercel-labs/skills --path skills/find-skills
```

---

### 6.7 安装 `security-threat-model`

```bash
python3 "$SKILL_INSTALLER" --repo openai/skills --path skills/.curated/security-threat-model
```

---

### 6.8 `pdf` 的说明

`pdf` 往往属于 OpenAI skills 环境里比较常见的一类。如果目标机器没有它，可以按下面方式安装：

```bash
python3 "$SKILL_INSTALLER" --repo openai/skills --path skills/.curated/pdf
```

## 7. 安装完成后要做什么

### 7.1 重启 Codex

这是最重要的一步。

新装好的 skill 往往要在 **重启 Codex** 后才会被新会话识别。

### 7.2 检查目录

你可以用下面命令确认 skill 目录是否已创建：

```bash
find ~/.codex/skills -maxdepth 1 -mindepth 1 -type d | sort
```

### 7.3 检查说明文件

每个正常安装的 skill 通常都会有 `SKILL.md`：

```bash
find ~/.codex/skills -maxdepth 2 -name SKILL.md | sort
```

## 8. 建议做一次功能验证

重启 Codex 后，建议做简单验证：

### 验证 `markitdown`

```text
请使用 markitdown 处理一个 PDF
```

### 验证 `playwright`

```text
请使用 playwright 打开一个网页并截图
```

### 验证 `security-threat-model`

```text
请使用 security-threat-model 审查一个本地 skill 目录
```

### 验证 `obsidian-markdown`

```text
请使用 obsidian-markdown 创建一篇 Obsidian 笔记
```

## 9. 常见问题

### 9.1 安装脚本不存在

如果这台机器上的 Codex 还没有 `.system/skill-installer`，说明当前环境和你现在这台机器不完全一致。

这时建议：

1. 先确认 Codex 是否完整安装
2. 先安装官方基础 skills
3. 或者先在 Codex 里让它帮你安装 `skill-installer`

### 9.2 提示目录已存在

说明这个 skill 之前已经装过。

处理方法：

- 想保留现状：跳过
- 想重装：先备份原目录，再删除旧目录，重新安装

例如：

```bash
mv ~/.codex/skills/markitdown ~/.codex/skills/markitdown.backup
```

### 9.3 安装成功但 Codex 不识别

最常见原因：

- 没有重启 Codex
- 装到了错误目录
- 当前会话还在使用旧的 skill 缓存

优先做法：

1. 检查 `~/.codex/skills/<skill-name>/SKILL.md` 是否存在
2. 完全退出并重启 Codex
3. 开一个新会话再测试

### 9.4 某些 skill 能安装但不能运行

这通常不是 skill 本身没装上，而是运行依赖没满足。

例如：

- `playwright` 需要 `npx`
- `defuddle` 依赖 Node.js 环境
- `obsidian-cli` 需要 Obsidian 正在运行
- `pdf` 可能需要 `poppler`

## 10. 最稳妥的恢复策略

如果你以后频繁换电脑，推荐固定用下面这套恢复流程：

1. 安装 Codex
2. 安装 Python、Git、Node.js
3. 克隆本仓库
4. 按本手册逐个安装 skills
5. 重启 Codex
6. 做 2 到 3 个关键 skill 验证

## 11. 你未来可以继续补充的内容

如果以后这套环境继续扩展，建议在仓库里再增加：

- `sources.md`
  - 记录每个 skill 的来源、替换历史和仓库地址
- `security-notes.md`
  - 记录哪些第三方 skill 做过安全审查
- `bootstrap.sh`
  - 把本手册里的安装命令做成一键安装脚本

这样以后换电脑会更快。
